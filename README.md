# Lo-Vibe

## 概要

Lo-Vibe は、Vibecodingをローカル環境で実施するためのツールです。本プロジェクトはそのソフトウェア開発を行うプロジェクトです。
Vibecodingとは、AIを活用してソフトウェア開発のプロセスを効率化する手法であり、プロジェクト管理、要件定義、設計、コーディング、テストなどの各フェーズで AI を活用します。

## 開発の進め方方針

- 基本的に作業はAIに指示して実行する。AIは開発者として扱う
- 本プロジェクトの目的は、Vibecodingをローカル環境で実施するためのツールを開発することであるため、Vibecodingのプロセスを実践することが重要である
- 開発手法は基本的にスクラム / MVP (Minimum Viable Product) の手法に則り進める
- タスク管理を GitHub Projects を使って行う。記載は API で実施
- ソースバージョン管理は Jujutsu を使用する
- 基本的にドキュメントは Markdown、図は Mermaid
  - ドキュメントはプロジェクトの全フェーズで重要な役割を果たすため、適切に管理することが重要である
  - ドキュメントはプロジェクトの全フェーズで更新されるため、ドキュメントの管理方法を明確にすることが重要である
- コーディング規約を定め、コードの品質を保つことが重要である

## ローカルに Vibecoding 環境を構築

### プロジェクト全体のディレクトリ構成概要

```text
lo-vibe/
├── README.md             # プロジェクト全体の概要と開発方針を記載
├── _document/            # ドキュメント全般を格納
│   ├── plan/              # プロジェクト計画関連ドキュメント
│   ├── requirement/       # 要求仕様関連ドキュメント
│   ├── definition/        # 要件定義関連ドキュメント
│   └── design/            # 設計関連ドキュメント
├── _compose/             # プロジェクトのシステム構成をコード化するディレクトリ
├── <サービス名>/           # ソースコード全般を格納
│   └── <サービス関連のリソース>  # サービス関連のリソース（例: API 定義、DB マイグレーションファイルなど）
├── test/                 # テストコード全般を格納
│   ├── unit/             # 単体テストコード
│   └── integration/      # 結合テストコード
└── tools/                # 開発・運用を効率化するツール類を格納
    ├── README.md         # ツールディレクトリの概要とツール一覧を記載
    └── save_promptinglog/ # プロンプトログ保存ツール (Go)
```



- ドキュメント
  - 要求仕様フェーズ
  - 要件定義フェーズ
    - プロジェクト計画
      - プロジェクト管理方法: Scrum
      - タスク管理: GitHub Projects をカンバン形式で管理
      - ソースバージョン管理: JuJutsu
      - JuJutsu を使った GitFlow を考える
    - 要件定義書
  - 設計フェーズ
    - 規約
      - コーディング規約
      - ドキュメント規約
    - システム構成
    - イテレーションロードマップ
- ソースコード
  - コーディング規約に従う
  - 単体テストコード
  - 結合テストコード
- ツール
  - プロンプトログ保存ツール (Go)
  - その他、開発・運用を効率化するツール類
