## 1. 対応番号

- 3章/4章の対応番号: 6

## 2. 主な機能

- ローカル LLM 推論サーバ
- Hugging Face 由来モデルの取得、配置、実行
- OpenAI 互換 API との連携
- GPU を使った低遅延推論

**利用観点**

- 主要ユースケース: コード生成、要約、RAG 回答生成をローカル環境で実行する場面
- 呼び出し目的: 外部 API 依存を抑えつつ、低遅延で再現可能な推論基盤を確保するため
- Output活用目的: 生成テキストと推論メトリクスを LiteLLM 経由で各クライアントへ返し、品質監視に繋げるため

## 3. 運用想定

- 実行場所: Linux サーバの ai ネットワーク
- 接続元: LiteLLM
- モデル供給元: Hugging Face（承認済みモデルリスト）
- リソース: GPU 優先、メモリとディスク容量を十分確保
- 保守: モデル更新時に精度・速度の回帰確認

## 5. 運用ルール

- 直接外部公開しない
- 用途別に利用モデルを固定し、評価結果を記録する
- モデルは Hugging Face のリビジョン固定で取得し、latest 運用を避ける
- ライセンス条件とハッシュを記録して監査可能にする
- モデルイメージと重みの保存容量を定期監視する
