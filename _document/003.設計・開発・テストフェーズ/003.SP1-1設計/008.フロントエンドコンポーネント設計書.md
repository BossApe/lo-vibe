## 3. SCR-001: overview/+page.svelte 詳細設計

### 3.1 責務

- システム概要テキストをユーザが入力できる UI を提供する
- フロントバリデーション（空文字・文字数超過）を実施する
- `POST /api/v1/system-overviews` を呼び出して DB に保存する
- 保存結果（成功 ID・エラーメッセージ）をユーザに表示する

---

### 3.2 状態変数（$state）

Svelte 5 のルーン構文（`$state`）を使用。

| 変数名 | 型 | 初期値 | 説明 |
| --- | --- | --- | --- |
| `content` | `string` | `''` | テキストエリアのバインド値（入力内容） |
| `errorMessage` | `string` | `''` | エラーメッセージ文字列。空文字ならエラー表示なし |
| `successMessage` | `string` | `''` | 成功メッセージ文字列。空文字なら表示なし |
| `isSubmitting` | `boolean` | `false` | API 呼び出し中フラグ。`true` の間ボタン・テキストエリアを無効化 |

---

### 3.3 定数

| 定数名 | 型 | 値 | 説明 |
| --- | --- | --- | --- |
| `API_BASE` | `string` | `import.meta.env.VITE_API_BASE_URL \|\| 'http://localhost:8080'` | API のベース URL |
| `MAX_LENGTH` | `number` | `4096` | コンテンツ最大文字数（API のバリデーションルールと一致させる） |

---

### 3.4 関数

#### `validate(): string`

送信前に呼ばれるバリデーション関数。

| 条件 | 返却値 |
| --- | --- |
| `content.trim()` が空 | `"システム概要を入力してください。"` |
| `content.length > MAX_LENGTH` | `"{MAX_LENGTH}文字以内で入力してください（現在: {N}文字）。"` |
| 問題なし | `""`（空文字） |

> **注意**: `content.trim()` で空白チェックしているが、実際の保存値は `content`（トリム前）を送信している。API 側のバリデーションは `content == ""` のみのため、空白のみの文字列は API に通過する場合がある。

---

#### `handleSubmit(e: Event): Promise<void>`

`<form onsubmit>` に紐づく非同期イベントハンドラ。

```
開始
  ├─ e.preventDefault()           # デフォルト送信防止
  ├─ errorMessage = ''            # 前回メッセージをクリア
  ├─ successMessage = ''
  ├─ validate() → エラーあり → errorMessage にセット → 終了
  ├─ isSubmitting = true
  ├─ try:
  │   ├─ fetch POST /api/v1/system-overviews
  │   ├─ res.status == 201 → successMessage にID表示・content をリセット
  │   ├─ res.status == 422 → errorMessage に API のメッセージを表示
  │   └─ その他 → errorMessage に固定メッセージを表示
  ├─ catch:
  │   └─ errorMessage に "ネットワークエラー" を表示
  └─ finally:
      └─ isSubmitting = false
```

---

### 3.5 HTML テンプレート構造

```
<svelte:head>
  <title>システム概要入力 | Musuhi</title>
</svelte:head>

<main.container>
  <h1>システム概要入力</h1>
  <p.description> 説明文 </p>

  <form onsubmit={handleSubmit}>
    <div.field>
      <label for="content"> ラベル <span.required>*</span> </label>
      <textarea
        id="content"
        bind:value={content}
        rows={12}
        maxlength={MAX_LENGTH}
        disabled={isSubmitting}
        aria-describedby="error-message"
        aria-invalid={!!errorMessage}
      />
      <div.char-count aria-live="polite"> N / 4096 文字 </div>
    </div>

    {#if errorMessage}
      <div#error-message.alert.alert-error role="alert"> {errorMessage} </div>
    {/if}

    {#if successMessage}
      <div.alert.alert-success role="status"> {successMessage} </div>
    {/if}

    <div.actions>
      <button type="submit" disabled={isSubmitting}>
        {isSubmitting ? '保存中...' : '保存する'}
      </button>
    </div>
  </form>
</main>
```

---

### 3.6 アクセシビリティ対応

| 対応 | 実装方法 |
| --- | --- |
| エラーと入力フォームの紐づけ | `aria-describedby="error-message"` |
| 入力値のバリデーション状態通知 | `aria-invalid={!!errorMessage}` |
| エラーメッセージのライブリージョン | `role="alert"` |
| 成功メッセージのライブリージョン | `role="status"` |
| 文字数カウントの読み上げ | `aria-live="polite"` |
| 送信中の操作禁止 | `disabled={isSubmitting}` |

---

### 3.7 スタイル

`<style>` はスコープ付き CSS（Svelte デフォルト）。

| セレクタ | 役割 |
| --- | --- |
| `.container` | `max-width: 720px` 中央配置 |
| `.description` | 説明文スタイル |
| `.field` | ラベル+テキストエリア+文字数のまとまり |
| `.required` | `*` マークを赤色表示 |
| `textarea` | フォント・パディング・ボーダー |
| `textarea:focus` | フォーカスリング（`#0066cc` アウトライン） |
| `textarea[aria-invalid='true']` | エラー時ボーダーを赤（`#c00`）に変更 |
| `.char-count` | 文字数カウント |
| `.alert` | 共通アラートベーススタイル |
| `.alert-error` | 赤系エラーアラート |
| `.alert-success` | 緑系成功アラート |
| `.actions` | ボタン配置 |
| `button` | 送信ボタン（無効時はグレー化） |

## 6. コンポーネント間データフロー

```mermaid
flowchart LR
    User([ユーザ]) -->|テキスト入力| TA[textarea<br>bind:value={content}]
    TA -->|content 更新| CV[char-count<br>リアルタイム表示]

    User -->|送信クリック| BTN[button<br>type=submit]
    BTN -->|onsubmit| HS[handleSubmit]

    HS -->|validate| VLD{バリデーション}
    VLD -->|NG| EM[errorMessage 表示]
    VLD -->|OK| API[POST /api/v1/system-overviews]

    API -->|201| SM[successMessage 表示<br>content リセット]
    API -->|422| EM2[errorMessage 表示<br>API メッセージ]
    API -->|その他| EM3[errorMessage 表示<br>固定メッセージ]
    API -->|例外| EM4[errorMessage 表示<br>ネットワークエラー]
```
