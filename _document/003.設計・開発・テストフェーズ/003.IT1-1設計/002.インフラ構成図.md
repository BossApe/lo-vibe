# IT1-1 インフラ構成図

[一覧](../README.md)

> 対象: IT1-1（FR-001〜FR-004）実装時点のインフラ構成

## 1. 全体構成図

```mermaid
flowchart LR
    Dev["開発者\n(ブラウザ / VS Code)"]

    subgraph DockerCompose["Docker Compose (ローカル開発環境)"]
        UI["musuhi-frontend\n(SvelteKit / Node)\n:5173"]
        API["musuhi-api\n(Go / net/http)\n:8080"]
        DB["PostgreSQL 16\n:5432\n(DB: musuhi)"]
    end

    GitHub["GitHub\n(REST API / GraphQL API)"]

    Dev -->|ブラウザアクセス| UI
    UI -->|POST /api/v1/system-overviews\nGET /api/v1/system-overviews/{id}| API
    API -->|SQL: INSERT / SELECT| DB
    API -->|POST /repos\nGraphQL: Projects v2| GitHub
```

## 2. IT1-1 対象サービス

| サービス | イメージ / 技術 | ポート | 役割 |
| --- | --- | --- | --- |
| musuhi-frontend | SvelteKit (Node 20) | 5173 | SCR-001〜SCR-004 の UI |
| musuhi-api | Go 1.22 / net/http | 8080 | FR-001〜FR-004 のビジネスロジック・REST API |
| PostgreSQL | postgres:16-alpine | 5432 | system_overviews / projects 等のデータ永続化 |

> IT1-1 スコープでは LiteLLM / Ollama / Traefik / 監視層は対象外とする。

## 3. API ルーティング（IT1-1 スコープ）

| メソッド | パス | 機能 | FR |
| --- | --- | --- | --- |
| GET | /health | ヘルスチェック | — |
| POST | /api/v1/system-overviews | システム概要を保存 | FR-001 |
| GET | /api/v1/system-overviews/{id} | システム概要を取得 | FR-001 |

## 4. データフロー（FR-001）

```
[ブラウザ]
  │  POST /api/v1/system-overviews  {"content":"..."}
  ▼
[musuhi-api :8080]
  │  handler → service → repository
  │  INSERT INTO system_overviews (content) VALUES ($1)
  ▼
[PostgreSQL :5432 / DB: musuhi]
  テーブル: system_overviews
  ┌──────────┬──────┬───────────────────────┐
  │ id (UUID)│content(TEXT)│created_at(TIMESTAMPTZ)│
  └──────────┴──────┴───────────────────────┘
```
