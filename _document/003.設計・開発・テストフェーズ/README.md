## 2. フェーズタスク設計案

本フェーズでは、GitHub Projects を以下の3階層で運用する。

- フェーズタスク: 期間管理と成果物管理の単位
- イテレーションタスク: 1〜2週で区切る推進単位
- チケットタスク: 1〜2日で完了する実行単位

詳細は [001-03.フェーズ・イテレーション・チケット設計規約](001.開発規約/001-03.フェーズ・イテレーション・チケット設計規約.md) を参照する。

## 3. 運用ルール要約

- 1チケットは 1成果物・1担当・1完了条件に分解する
- 依存関係は `Depends on` で必ず明示する
- ガントチャート確認は Roadmap ビューを正とする
- 各イテレーション完了時にレビュー記録を残す
