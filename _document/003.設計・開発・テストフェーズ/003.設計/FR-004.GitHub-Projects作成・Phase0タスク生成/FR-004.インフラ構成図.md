> 対象: TK1-1-4（FR-004）実装時点のインフラ構成

## 1. 全体構成図

```mermaid
flowchart LR
  Dev["開発者<br/>ブラウザ / VS Code"]

  subgraph DockerCompose["Docker Compose (ローカル開発環境)"]
    UI["musuhi-frontend<br/>SvelteKit / Node<br/>:3000 (Docker) / :5173 (dev)"]
    API["musuhi-api<br/>Go / net/http<br/>:8080"]
  end

  GitHub["GitHub<br/>GraphQL API"]

  Dev -->|ブラウザアクセス| UI
  UI -->|POST /api/v1/projects/{id}/github-projects<br/>POST /api/v1/projects/{id}/phase0-tasks| API
  API -->|GraphQL mutation/query| GitHub
```

## 2. 対象サービス

| サービス | イメージ / 技術 | ポート | 役割 |
| --- | --- | --- | --- |
| musuhi-frontend | SvelteKit (Node 20) | 3000 (Docker) / 5173 (dev) | SCR-004 の UI |
| musuhi-api | Go 1.22 / net/http | 8080 | FR-004 のビジネスロジック・REST API |
| GitHub | GraphQL API | 443 (外部) | Projects v2 作成・タスク登録 |

> SP1-1 スコープでは 通知連携やレポート自動配信は対象外とする。

## 3. API ルーティング（機能内スコープ）

| メソッド | パス | 機能 | FR |
| --- | --- | --- | --- |
| POST | /api/v1/projects/{id}/github-projects | Projects v2 ボード作成 | FR-004 |
| POST | /api/v1/projects/{id}/phase0-tasks | Phase0 固定タスク生成・登録 | FR-004 |

## 4. データフロー（FR-004）

```
[ブラウザ]
  │  POST /api/v1/projects/{id}/github-projects {owner, title}
  ▼
[musuhi-api :8080]
  │  handler → service
  │  createProjectV2 mutation 実行
  ▼
[GitHub GraphQL API]
  │  projectsId / projectsUrl を返却
  ▼
[musuhi-api :8080]
  │  POST /api/v1/projects/{id}/phase0-tasks {owner, projectsId}
  │  addProjectV2DraftIssue / updateProjectV2ItemFieldValue を順次実行
  ▼
[GitHub Projects v2]
```
