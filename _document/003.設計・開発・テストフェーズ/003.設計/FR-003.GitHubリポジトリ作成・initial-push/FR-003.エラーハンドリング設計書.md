## 1. エラー処理方針

| 方針 | 内容 |
| --- | --- |
| ユーザ向け表示 | API から返る `error.message` を表示し、内部スタックは非表示 |
| 分類 | `ErrValidation` とそれ以外を分離して HTTP へマッピング |
| 連携失敗 | Forgejo/`git` 実行失敗は `INTERNAL_ERROR` として扱う |

## 2. レイヤー別責務

### 2.1 Handler 層

| 条件 | HTTP | code |
| --- | --- | --- |
| JSON 形式不正 | 400 | `BAD_REQUEST` |
| Service が `ErrValidation` | 422 | `VALIDATION_ERROR` |
| その他 | 500 | `INTERNAL_ERROR` |

### 2.2 Service 層

| 条件 | 返却エラー |
| --- | --- |
| 必須項目不足・形式不正 | `ErrValidation` |
| Forgejo入力系既知エラー（重複・認証） | `ErrValidation` |
| `gh` / `git` 実行失敗 | ラップして返却（Handler で 500） |

## 3. エラーコード一覧

| エラーコード | HTTP | 例 |
| --- | --- | --- |
| `BAD_REQUEST` | 400 | JSON 形式不正 |
| `VALIDATION_ERROR` | 422 | owner 空、repoName 不正、visibility 不正 |
| `INTERNAL_ERROR` | 500 | Forgejo API失敗、git push失敗 |

## 4. 既知課題

| # | 課題 | 対応方針 |
| --- | --- | --- |
| 1 | Forgejo API の詳細エラーコード未分類 | 将来 `UNAUTHORIZED` / `CONFLICT` などへ分岐追加 |
