# 007.エラーハンドリング設計書

前: [006.テスト仕様書](006.テスト仕様書.md) | [一覧](README.md) | 次: [008.フロントエンドコンポーネント設計書](008.フロントエンドコンポーネント設計書.md)

## 1. レイヤー別責務

- Handler: JSON 形式不正を BAD_REQUEST で返す
- Service: 入力不正を VALIDATION_ERROR に正規化
- External Client: GraphQL 実行失敗をエラーとして返却

## 2. エラー一覧

| レイヤー | 条件 | HTTP | code |
| --- | --- | --- | --- |
| Handler | JSON decode 失敗 | 400 | BAD_REQUEST |
| Service | owner/title/projectsId が空 | 422 | VALIDATION_ERROR |
| Client | gh api graphql 実行失敗 | 500 | INTERNAL_ERROR |
| Client | GraphQL errors が返却 | 500 | INTERNAL_ERROR |

## 3. ログ方針

- 外部コマンド失敗時は標準出力/標準エラーを含めてラップ
- シークレット値はログへ出力しない
