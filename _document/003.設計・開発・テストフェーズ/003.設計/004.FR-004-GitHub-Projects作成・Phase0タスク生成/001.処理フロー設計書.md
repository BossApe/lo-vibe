# 001.処理フロー設計書

前: [README](README.md) | [一覧](README.md) | 次: [003.API仕様書](003.API仕様書.md)

## 1. 目的

FR-004 として GitHub Projects v2 ボードの作成、および Phase0 固定タスク登録処理のサーバー側フローを定義する。

## 2. 対象 API

- POST /api/v1/projects/{id}/github-projects
- POST /api/v1/projects/{id}/phase0-tasks

## 3. 正常系フロー

1. リクエスト受信（owner/title または owner/projectsId）
2. Handler で JSON 形式を検証
3. Service で必須入力を検証
4. GitHub GraphQL API を実行
5. 作成結果を整形して 201 を返却

## 4. Projects 作成フロー

1. owner を User/Organization として解決
2. createProjectV2 mutation を実行
3. Type/Phase/Sprint/Priority/Estimate/Depends on フィールドを追加
4. projectsUrl, projectsId, status を返却

## 5. Phase0 タスク登録フロー

1. プロジェクトフィールド定義を取得
2. 固定タスク定義（PH0, SP0-1〜SP0-4, TK0-1-1〜TK0-4-2）を順次処理
3. addProjectV2DraftIssue mutation でアイテム追加
4. updateProjectV2ItemFieldValue mutation で Type/Phase/Sprint/Priority/Estimate を設定
5. tasks と status を返却

## 6. 異常系フロー

- 入力不備: VALIDATION_ERROR（422）
- GraphQL 実行失敗: INTERNAL_ERROR（500）
- GraphQL の errors 返却: INTERNAL_ERROR（500）
