# FR-003-テスト仕様書

前: [FR-003-画面設計書](FR-003-画面設計書.md) | [一覧](README.md) | 次: [FR-003-エラーハンドリング設計書](FR-003-エラーハンドリング設計書.md)

> 対象: TK1-1-3 で実装した FR-003「GitHubリポジトリ作成・initial push」の単体/内部結合/外部結合テスト仕様

## 1. テスト方針

| 方針 | 内容 |
| --- | --- |
| 単体テスト | Service 層・Handler 層をモックで検証 |
| 内部結合テスト | `httptest.Server` による API レイヤ連結確認 |
| 外部結合テスト | GitHub 実接続（副作用あり）を専用 owner で実施 |

## 2. テストケース一覧

### 2.1 単体テスト（Service/Handler）

| TC-ID | 観点 | テスト内容 | 期待結果 |
| --- | --- | --- | --- |
| TC-FR003-UT-N01 | 正常系 | 有効入力で CreateRepositoryWithExternal を実行 | `pushStatus=success` |
| TC-FR003-UT-E01 | 異常系 | owner 空で実行 | `VALIDATION_ERROR` |
| TC-FR003-UT-E02 | 異常系 | localPath 相対パスで実行 | `VALIDATION_ERROR` |
| TC-FR003-UT-E03 | 異常系 | リポジトリ名重複エラー | `VALIDATION_ERROR` |

### 2.2 API/内部結合テスト

| TC-ID | 観点 | テスト内容 | 期待結果 |
| --- | --- | --- | --- |
| TC-FR003-IT-N01 | 正常系 | `POST /api/v1/projects/with-external` | 201 / `repositoryUrl` 返却 |
| TC-FR003-IT-E01 | 異常系 | owner を空で実行 | 422 / `VALIDATION_ERROR` |

### 2.3 外部結合テスト

| TC-ID | 観点 | テスト内容 | 期待結果 |
| --- | --- | --- | --- |
| TC-FR003-EXT-N01 | 連携 | GitHub repo 作成＋push 実行 | GitHub 上に実リポジトリが存在 |
| TC-FR003-EXT-E01 | 異常系 | 認証不備で実行 | 500 または認証エラー記録 |

## 3. 実行コマンド

```bash
# 単体
cd services/musuhi-api/src
go test ./internal/handler ./internal/service -run 'WithExternal|CreateRepositoryWithExternal' -v

# API / 内部結合
go test ./test/integration -run WithExternal -v

# 外部結合（実接続）
RUN_EXTERNAL_TESTS=1 GH_TOKEN=<token> go test ./test/external -run WithExternal -v
```
