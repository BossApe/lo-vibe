## 1. POST /api/v1/projects/with-external

### 概要

GitHub に新規リポジトリを作成し、ローカルプロジェクトを初回 commit/push する。

### リクエスト

```json
{
  "owner": "BossApe",
  "repoName": "demo-project",
  "visibility": "private",
  "localPath": "/Users/yourname/gitspace/demo-project",
  "commitMessage": "initial commit"
}
```

### リクエストパラメータ制約

| フィールド | 制約 |
| --- | --- |
| `owner` | 必須。GitHub owner（ユーザまたは組織） |
| `repoName` | 必須。先頭は英数字。以降は英数字・`_`・`-` のみ。パターン: `^[a-zA-Z0-9][a-zA-Z0-9_-]*$` |
| `visibility` | 必須。`public` または `private` |
| `localPath` | 必須。絶対パスで、既存ディレクトリであること |
| `commitMessage` | 必須。1〜256文字 |

### レスポンス (201)

```json
{
  "data": {
    "repositoryUrl": "https://github.com/BossApe/demo-project",
    "externalProjectId": "123456",
    "pushStatus": "success"
  }
}
```

### エラーコード

| コード | HTTP | 条件 |
| --- | --- | --- |
| `BAD_REQUEST` | 400 | JSON 形式不正 |
| `VALIDATION_ERROR` | 422 | owner/repoName/visibility/localPath/commitMessage の不正、または GitHub 側入力エラー |
| `NOT_FOUND` | 404 | 該当なし |
| `INTERNAL_ERROR` | 500 | GitHub API 実行失敗、git 操作失敗などのサーバ内部エラー |

### エラーレスポンス形式

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "owner is required"
  }
}
```
