## 1. 使い方

- 設計書レビュー時に、実装（API/画面/テスト）を正として確認する
- 対応済みは `[x]`、要確認は `[ ]`、非対象は `N/A` を記録する
- 必要に応じて「確認結果メモ」に根拠ファイルを追記する

## 2. API 観点

- [ ] エンドポイント一覧が実装と一致している
- [ ] HTTP メソッド（GET/POST/PUT/PATCH/DELETE）が実装と一致している
- [ ] リクエスト JSON の必須/任意が実装と一致している
- [ ] レスポンス JSON の項目名と型が実装と一致している
- [ ] ステータスコード（200/201/400/422/404/500）が実装と一致している
- [ ] バリデーション条件（ID/必須項目/形式制約）が実装と一致している
- [ ] 機能固有ルール（候補生成、連携条件、状態遷移など）が記載されている

## 3. 画面 観点

- [ ] 画面 URL と主要アクションが実装と一致している
- [ ] URL パラメータ未指定時や初期表示時の挙動が記載されている
- [ ] 画面状態変数（loading、error、success、選択状態など）が記載されている
- [ ] API 呼び出しタイミング（onMount、イベント時、再実行条件）が記載されている
- [ ] 入力制御・バリデーション・ボタン活性条件が記載されている
- [ ] エラーメッセージ表示仕様（APIメッセージ優先/固定文言）が記載されている
- [ ] 環境変数依存（API_BASE 等）と実行環境差分（compose など）が記載されている

## 4. テスト 観点

- [ ] Service 単体テストの主要ケースが実装と一致している
- [ ] Handler 単体テストの主要ケースが実装と一致している
- [ ] 内部結合テストの主要ケースが実装と一致している
- [ ] 必要に応じて外部結合/E2E/受け入れテスト観点が記載されている
- [ ] 正常系・異常系・境界値の観点が欠落していない
- [ ] テストケース番号の重複や欠番がない
- [ ] AC 対応表のテスト番号・範囲が最新化されている

## 5. 異常系 観点

- [ ] BAD_REQUEST の発生条件と表示メッセージが実装と一致している
- [ ] VALIDATION_ERROR の発生条件と表示メッセージが実装と一致している
- [ ] NOT_FOUND の発生条件と表示メッセージが実装と一致している
- [ ] INTERNAL_ERROR の発生条件と表示メッセージが実装と一致している
- [ ] ネットワーク断時のフロント挙動（catch/finally）が記載されている
- [ ] ログ方針（アクセスログ/エラーログ）が記載されている

## 6. 変更時チェック（差分レビュー用）

- [ ] 実装変更ファイルの差分を確認した
- [ ] 影響する設計書を特定して更新した
- [ ] 影響するテスト仕様書を更新した
- [ ] README のリンク一覧を更新した
- [ ] 更新履歴を必要に応じて追記した

### 7.1 設計（ドキュメント）準拠

- [ ] API 設計が 001-05 のレイヤ責務・ルーティング規約・レスポンス規約に準拠している
- [ ] 画面設計が 001-04 の状態管理・API通信・エラーハンドリング規約に準拠している
- [ ] テスト仕様が 001-01 / 001-05 のテスト命名・配置・観点規約に準拠している
- [ ] エラーハンドリング設計が 001-05 / 001-03 のエラー分類・ログ運用規約に準拠している

### 7.2 開発（実装）準拠

- [ ] バックエンド実装が 001-01 / 001-05 の依存方向（handler -> service -> repository）と命名規約に準拠している
- [ ] フロントエンド実装が 001-04 のスクリプト構成順・状態命名・API呼び出し規約に準拠している
- [ ] 機密情報の取り扱いが 001-01 のセキュリティ規約に準拠している（コード直書き禁止）
- [ ] ログ出力が 001-03 のログレベル・マスキング・出力項目規約に準拠している
- [ ] 補助ツールや運用スクリプトを追加した場合、001-02 の配置・README記載規約に準拠している

### 7.3 各種テスト準拠

- [ ] 単体テストが 001-01 / 001-05 の命名規約・モック方針・配置規約に準拠している
- [ ] 内部結合テストが 001-01 / 001-05 の配置規約・観点規約に準拠している
- [ ] 外部結合テスト/受け入れテスト/E2E テストの追加時に、001-01 のテスト種別分離規約に準拠している
- [ ] テスト実行・カバレッジ確認手順が 001-01 / 001-05 の規約に沿って記載されている
