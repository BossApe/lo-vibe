## 1. エラー処理方針

| 方針 | 内容 |
| --- | --- |
| ユーザ向けメッセージ | 日本語で分かりやすく表示する。内部エラー詳細はユーザに露出しない |
| センチネルエラー | `ErrValidation` / `ErrNotFound` で分岐し、HTTP ステータスにマッピングする |
| エラーのラッピング | `fmt.Errorf("%w: ...", ErrXxx)` でラップし `errors.Is()` で判定する |
| パニック防止 | Handler 層で全エラーをキャッチし、500 を返す。パニックは上位ミドルウェアで回収（未実装・将来対応） |
| ログ出力 | SP1-1 では Logger ミドルウェアでリクエストログを出力。エラーログは未実装（将来対応） |

## 2. エラー分類

| カテゴリ | 説明 | 例 |
| --- | --- | --- |
| **クライアントエラー（入力）** | ユーザの入力内容が正しくない | 空文字・文字数超過・UUID 形式不正 |
| **クライアントエラー（形式）** | リクエスト形式が不正 | JSON デコード失敗 |
| **リソースエラー** | 対象リソースが存在しない | 指定 ID の `system_overviews` レコードなし |
| **インフラエラー** | DB 接続失敗・タイムアウト等 | PostgreSQL 接続不可 |
| **ネットワークエラー** | フロント→API の通信断 | fetch の例外 |

## 3. レイヤー別エラー処理責務

### 3.1 フロントエンド（SvelteKit）

**ファイル**: `services/musuhi-frontend/src/routes/overview/+page.svelte`

| エラー種別 | 処理タイミング | 処理内容 | 表示メッセージ |
| --- | --- | --- | --- |
| 空文字入力 | 送信前（フロントバリデーション） | `errorMessage` にセット・送信中止 | `システム概要を入力してください。` |
| 文字数超過 | 送信前（フロントバリデーション） | `errorMessage` にセット・送信中止 | `4096文字以内で入力してください（現在: {N}文字）。` |
| API 422 | fetch 後 | `error.message` を `errorMessage` にセット | API レスポンスの `error.message` をそのまま表示 |
| API 4xx / 5xx（422以外） | fetch 後 | 固定メッセージを `errorMessage` にセット | `サーバーエラーが発生しました。しばらく待ってから再試行してください。` |
| ネットワーク断（fetch 例外） | catch 句 | 固定メッセージを `errorMessage` にセット | `ネットワークエラーが発生しました。接続を確認してください。` |

**実装上の注意:**
- エラー発生時は `isSubmitting = false` に必ず戻す（`finally` 句で保証）
- 再送信時は `errorMessage = ''` / `successMessage = ''` をクリアしてから処理開始

### 3.2 Handler 層（Go）

**ファイル**: `services/musuhi-api/src/internal/handler/system_overview.go`

| エラー種別 | 判定方法 | HTTP ステータス | エラーコード |
| --- | --- | --- | --- |
| JSON デコード失敗 | `json.Decode` エラー | 400 | `BAD_REQUEST` |
| バリデーションエラー | `errors.Is(err, service.ErrValidation)` | 422 | `VALIDATION_ERROR` |
| リソース未存在 | `errors.Is(err, service.ErrNotFound)` | 404 | `NOT_FOUND` |
| UUID 形式不正（GET/PUT） | `errors.Is(err, service.ErrValidation)` | 422 | `VALIDATION_ERROR` |
| PathValue 空（GET/PUT） | `id == ""` チェック | 400 | `BAD_REQUEST` |
| その他（DB エラー等） | 上記以外のエラー | 500 | `INTERNAL_ERROR` |

**設計上の注意:**
- 500 エラーの場合、内部エラー詳細は `message` に含めない（`"サーバーエラーが発生しました"` 固定）
- `details` フィールドは VALIDATION_ERROR のみ付与する

---

### 3.3 Service 層（Go）

**ファイル**: `services/musuhi-api/src/internal/service/system_overview.go`

| エラー種別 | 発生条件 | 返却エラー |
| --- | --- | --- |
| コンテンツ空文字 | `content == ""` | `fmt.Errorf("%w: content is required", ErrValidation)` |
| コンテンツ文字数超過 | `utf8.RuneCountInString(content) > 4096` | `fmt.Errorf("%w: content must be 4096 characters or less", ErrValidation)` |
| UUID 形式不正 | `uuid.Parse(rawID)` 失敗 | `fmt.Errorf("%w: invalid id format", ErrValidation)` |
| レコード未存在（取得・更新共通） | `pgx.ErrNoRows` | `fmt.Errorf("%w: system_overview id=%s", ErrNotFound, rawID)` |
| DB エラー（その他） | repo エラー | `fmt.Errorf("systemOverviewService.Create/Update: %w", err)` |

**センチネルエラー定義:**

```go
var ErrNotFound  = errors.New("not found")
var ErrValidation = errors.New("validation error")
```

---

### 3.4 Repository 層（Go）

**ファイル**: `services/musuhi-api/src/internal/repository/system_overview_postgres.go`

| エラー種別 | 発生条件 | 返却エラー |
| --- | --- | --- |
| レコード未存在 | `pgx.ErrNoRows` | `fmt.Errorf("systemOverviewRepository.FindByID: %w", err)` |
| レコード更新失敗（未存在） | `pgx.ErrNoRows`（UPDATE 後） | `fmt.Errorf("systemOverviewRepository.UpdateByID: %w", err)` |
| DB 接続エラー | pgxpool エラー | `fmt.Errorf("systemOverviewRepository.Create: %w", err)` |

**注意:** Repository 層は `ErrNotFound` / `ErrValidation` を使用しない。エラー種別の判定は Service 層が行う。

## 4. エラーコード一覧

| エラーコード | HTTP | 発生レイヤー | 説明 |
| --- | --- | --- | --- |
| `BAD_REQUEST` | 400 | Handler | JSON 形式不正 / UUID 形式不正 / PathValue 欠如 |
| `VALIDATION_ERROR` | 422 | Handler（Service 判定） | バリデーションルール違反（空文字・文字数超過） |
| `NOT_FOUND` | 404 | Handler（Service 判定） | 指定 ID のリソースが存在しない |
| `INTERNAL_ERROR` | 500 | Handler | DB エラー・予期しないエラー |

## 6. ログ出力方針

**現状（SP1-1 実装済み）:**

| ログ種別 | 出力場所 | 内容 | ファイル |
| --- | --- | --- | --- |
| リクエストログ | stderr | `METHOD PATH DURATION STATUS` | `middleware/logger.go` |

**未実装（今後対応予定）:**

| ログ種別 | 優先度 | 内容 |
| --- | --- | --- |
| エラーログ | 高 | `INTERNAL_ERROR` 発生時にスタックトレースを出力 |
| バリデーションログ | 低 | 422 エラー時のフィールド・値を記録（デバッグ用） |
| パニックリカバリログ | 高 | ミドルウェアでパニックを回収してログ出力 |

## 7. 既知の課題・今後の対応

| # | 課題 | 優先度 | 対応予定 |
| --- | --- | --- | --- |
| 1 | 空白のみ文字列（`"   "`）がバリデーションを通過する（`content == ""` のみチェック） | 中 | `strings.TrimSpace(content) == ""` に修正を検討 |
| 2 | 500 エラー時の内部エラーがログに出力されない | 高 | エラーログミドルウェアを追加する |
| 3 | パニック時のリカバリ機構がない | 高 | `recover()` を使用した middleware を追加する |
| 4 | フロントエンドで 422 以外の 4xx（400・404）が `サーバーエラー` と表示される | 低 | ステータスコード別のメッセージ分岐を追加する |
