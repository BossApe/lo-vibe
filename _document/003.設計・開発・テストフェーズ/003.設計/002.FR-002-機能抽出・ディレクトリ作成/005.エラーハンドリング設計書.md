## 1. エラー処理方針

| 方針 | 内容 |
| --- | --- |
| ユーザ向けメッセージ | 日本語で分かりやすく表示する。内部エラー詳細はユーザに露出しない |
| センチネルエラー | `ErrValidation` / `ErrNotFound` で分岐し、HTTP ステータスにマッピングする |
| エラーのラッピング | `fmt.Errorf("%w: ...", ErrXxx)` でラップし `errors.Is()` で判定する |
| パニック防止 | Handler 層で全エラーをキャッチし、500 を返す。パニックは上位ミドルウェアで回収（未実装・将来対応） |
| ログ出力 | SP1-1 では Logger ミドルウェアでリクエストログを出力。エラーログは未実装（将来対応） |

---

## 2. エラー分類

| カテゴリ | 説明 | 例 |
| --- | --- | --- |
| **クライアントエラー（入力）** | ユーザの入力内容が正しくない | 空 overviewId・パターン違反のプロジェクト名・相対パス |
| **クライアントエラー（形式）** | リクエスト形式が不正 | JSON デコード失敗 |
| **リソースエラー** | 対象リソースが存在しない | 指定 UUID の `system_overviews` レコードなし |
| **インフラエラー** | ディレクトリ作成失敗・DB 接続障害等 | `os.MkdirAll` 失敗・PostgreSQL 接続不可 |
| **ネットワークエラー** | フロント→API の通信断 | `fetch` の例外 |

## 3. レイヤー別エラー処理責務

### 3.1 フロントエンド（SvelteKit）

**ファイル**: `services/musuhi-frontend/src/routes/projects/setup/+page.svelte`

| エラー種別 | 処理タイミング | 処理内容 | 表示メッセージ |
| --- | --- | --- | --- |
| API 422 / 404 | fetch 後 | `error.message` を `errorMessage` にセット | API レスポンスの `error.message` をそのまま表示 |
| API 4xx / 5xx（422・404 以外） | fetch 後 | 固定メッセージを `errorMessage` にセット | `"APIエラーが発生しました。"` |
| ネットワーク断（fetch 例外） | catch 句 | 固定メッセージを `errorMessage` にセット | `"処理に失敗しました。"` / `"初期ディレクトリ作成に失敗しました。"` |

**実装上の注意:**
- エラー発生時は `isLoading = false` に必ず戻す（`finally` 句で保証）
- 再実行時は `errorMessage = ''` / `directoryStatus = ''` をクリアしてから処理開始

---

### 3.2 Handler 層（Go）

**ファイル**: `services/musuhi-api/src/internal/handler/project.go`

| エラー種別 | 判定方法 | HTTP ステータス | エラーコード |
| --- | --- | --- | --- |
| JSON デコード失敗 | `json.Decode` エラー | 400 | `BAD_REQUEST` |
| バリデーションエラー | `errors.Is(err, service.ErrValidation)` | 422 | `VALIDATION_ERROR` |
| リソース未存在 | `errors.Is(err, service.ErrNotFound)` | 404 | `NOT_FOUND` |
| その他（DB エラー・OS エラー等） | 上記以外のエラー | 500 | `INTERNAL_ERROR` |

**設計上の注意:**
- 500 エラーの場合、内部エラー詳細は `message` に含めない（`"サーバーエラーが発生しました"` 固定）

---

### 3.3 Service 層（Go）

**ファイル**: `services/musuhi-api/src/internal/service/project.go`

#### ExtractFeatures / SuggestName 共通（loadOverviewContent）

| エラー種別 | 発生条件 | 返却エラー |
| --- | --- | --- |
| overviewId が UUID 形式不正 | `uuid.Parse` 失敗 | `fmt.Errorf("%w: invalid overviewId format", ErrValidation)` |
| レコード未存在 | `pgx.ErrNoRows` | `fmt.Errorf("%w: system_overview id=%s", ErrNotFound, overviewID)` |
| DB エラー（その他） | repo エラー | `fmt.Errorf("projectService.loadOverviewContent: %w", err)` |

#### InitDirectory

| エラー種別 | 発生条件 | 返却エラー |
| --- | --- | --- |
| projectName が空 | `strings.TrimSpace(projectName) == ""` | `fmt.Errorf("%w: projectName is required", ErrValidation)` |
| projectName がパターン違反 | `projectNamePattern.MatchString` 失敗 | `fmt.Errorf("%w: projectName must match %s", ErrValidation, ...)` |
| localPath が空 | `strings.TrimSpace(localPath) == ""` | `fmt.Errorf("%w: localPath is required", ErrValidation)` |
| localPath が相対パス | `!filepath.IsAbs(localPath)` | `fmt.Errorf("%w: localPath must be absolute path", ErrValidation)` |
| template が不正 | `template != "default"` | `fmt.Errorf("%w: unsupported template", ErrValidation)` |
| ディレクトリ作成失敗 | `os.MkdirAll` エラー | `fmt.Errorf("projectService.InitDirectory: %w", err)` |

## 4. エラーコード一覧

| エラーコード | HTTP | 意味 | 発生例 |
| --- | --- | --- | --- |
| `BAD_REQUEST` | 400 | JSON 形式不正 | `invalid-json` をリクエスト body に送信 |
| `VALIDATION_ERROR` | 422 | バリデーション違反 | 空 overviewId・パターン違反プロジェクト名・相対パス |
| `NOT_FOUND` | 404 | リソース未存在 | 削除済み UUID を overviewId に指定 |
| `INTERNAL_ERROR` | 500 | サーバー内部エラー | DB 障害・ディレクトリ作成失敗 |

## 5. ログ出力方針

| 項目 | 内容 |
| --- | --- |
| アクセスログ | Logger ミドルウェア（`internal/middleware/logger.go`）がリクエスト単位で stderr に出力 |
| エラーログ | SP1-1 スコープでは未実装。将来的に構造化ログ（`slog`）で追加予定 |

## 6. 既知の課題・今後の対応

| # | 課題 | 対応方針 |
| --- | --- | --- |
| 1 | ディレクトリ既存時の冪等性 | `os.MkdirAll` のため既存でも成功するが、ファイル競合は未考慮。将来的に上書き確認 UI を追加予定 |
| 2 | パニックリカバリ | Handler 層のパニックはミドルウェアで回収する仕組みが未実装。将来的に `recovery` ミドルウェアを追加予定 |
| 3 | エラーログ詳細化 | 500 エラー時のサーバーサイドログが `Logger` ミドルウェアのリクエストログのみ。将来的にエラーレベルのログを追加予定 |
