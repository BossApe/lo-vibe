## 3. SCR-002: projects/setup/+page.svelte 詳細設計

### 3.1 責務

- URL クエリパラメータ `overviewId` からシステム概要 ID を取得する
- マウント時に自動で機能抽出・プロジェクト名候補生成 API を連続呼び出しする
- プロジェクト名をコンボボックス（`<input list>`）で選択または自由入力できる
- AI 提案候補を選択した場合は選定理由を候補名の下に表示する
- ローカル作成先パスを自動補完（`BASE_DIR + selectedProjectName`）し、手動編集後は上書きしない
- 「初期ディレクトリ作成」ボタンで `POST /api/v1/projects/init-directory` を呼び出す
- 「概要入力に戻る」リンクで `overviewId` を保持したまま SCR-001 に戻る

---

### 3.2 型定義

```typescript
type ProjectNameCandidate = {
  name: string;
  reason?: string;
  aiSuggested: boolean;
};
```

---

### 3.3 定数

| 定数名 | 値 | 説明 |
| --- | --- | --- |
| `API_BASE` | `import.meta.env.VITE_API_BASE_URL ?? 'http://localhost:8080'` | API ベース URL |
| `BASE_DIR` | `'/Users/m.nohara/gitspace/'` | localPath 自動補完用のベースディレクトリ |

---

### 3.4 状態変数（$state）

Svelte 5 のルーン構文（`$state`）を使用。

| 変数名 | 型 | 初期値 | 説明 |
| --- | --- | --- | --- |
| `overviewId` | `string` | `''` | URL から取得したシステム概要 ID |
| `localPath` | `string` | `''` | 初期ディレクトリ作成先パス（自動補完 or 手動入力） |
| `localPathEdited` | `boolean` | `false` | ユーザが `localPath` を手動編集したかのフラグ |
| `selectedProjectName` | `string` | `''` | コンボボックスで選択・入力中のプロジェクト名 |
| `features` | `string[]` | `[]` | 抽出された機能一覧 |
| `components` | `string[]` | `[]` | 抽出された構成要素一覧 |
| `candidates` | `string[]` | `[]` | プロジェクト名候補の名前配列 |
| `candidateItems` | `ProjectNameCandidate[]` | `[]` | プロジェクト名候補の詳細情報配列 |
| `directoryStatus` | `string` | `''` | `init-directory` API の結果ステータス |
| `errorMessage` | `string` | `''` | エラーメッセージ表示用 |
| `isLoading` | `boolean` | `false` | API 呼び出し中フラグ |
| `hasAutoExtracted` | `boolean` | `false` | 自動機能抽出を実行済みかのフラグ（二重実行防止） |

---

### 3.5 派生値（$derived）

| 変数名 | 型 | 説明 |
| --- | --- | --- |
| `selectedCandidate` | `ProjectNameCandidate \| undefined` | `selectedProjectName` に一致する `candidateItems` の要素 |

---

### 3.6 副作用（$effect / onMount）

| 種別 | トリガー | 処理内容 |
| --- | --- | --- |
| `onMount` | コンポーネントマウント時 | URL クエリ `overviewId` を `overviewId` 状態変数にセット |
| `$effect` (localPath) | `localPathEdited` が false かつ `selectedProjectName` 変更時 | `localPath = BASE_DIR + selectedProjectName` に自動更新 |
| `$effect` (autoExtract) | `hasAutoExtracted` が false かつ `overviewId` が非空時 | `hasAutoExtracted = true` → `handleExtract()` を呼び出す（一度のみ） |

---

### 3.7 関数

| 関数名 | 用途 | 処理フロー |
| --- | --- | --- |
| `postJson(path, body)` | API POST ヘルパー | `fetch` → JSON パース → `res.ok` でなければ `error.message` を throw |
| `handleExtract()` | 機能抽出・プロジェクト名候補取得 | `extract-features` → `suggest-name` を順次 POST → 状態更新 |
| `handleInitDirectory()` | 初期ディレクトリ作成 | `init-directory` を POST → `directoryStatus` を更新 |

---

### 3.8 HTML テンプレート構造

```
<svelte:head> タイトル
<main.container>
  <p.back-link> 概要入力に戻るリンク（/overview?overviewId={overviewId}）
  <h1> プロジェクト名・構成要素確認
  <p> 説明テキスト

  {#if isLoading && features/candidates 未取得}
    <div.card> ローディングメッセージ
  {/if}

  {#if features.length > 0}
    <div.card>
      <h2> 抽出された機能
      <ul> {#each features}
      <h2> 構成要素
      <ul> {#each components}
  {/if}

  {#if candidates.length > 0}
    <div.card>
      <h2> プロジェクト名候補
      <label> プロジェクト名（コンボボックス）
      <input list="project-name-candidates" bind:value={selectedProjectName}>
      <datalist id="project-name-candidates"> {#each candidates}
      {#if selectedCandidate?.aiSuggested && reason}
        <p.candidate-reason> 選定理由テキスト
      {/if}
      <label> ローカル作成先パス
      <input bind:value={localPath} oninput={() => localPathEdited = true}>
      <button onclick={handleInitDirectory}> 初期ディレクトリ作成
  {/if}

  {#if directoryStatus}
    <p.success> ディレクトリ作成結果
  {/if}

  {#if errorMessage}
    <p.error role="alert"> エラーメッセージ
  {/if}
```

---

### 3.9 アクセシビリティ対応

| 対応内容 | 実装箇所 |
| --- | --- |
| `<label for>` によるフォーム関連付け | `projectName`・`localPath` 入力フィールド |
| `role="alert"` によるエラー読み上げ | `errorMessage` 表示段落 |
| `disabled` 制御による誤送信防止 | `初期ディレクトリ作成` ボタン（`isLoading` または `selectedProjectName`/`localPath` が空の場合） |

---

### 3.10 スタイル

| クラス | 用途 |
| --- | --- |
| `.container` | 最大幅 840px・中央揃え・padding |
| `.card` | ボーダー・角丸・flex 縦並び（gap 0.6rem） |
| `.back-link` | 戻るリンクのマージン調整 |
| `input` | padding・border・角丸 |
| `.candidate-reason` | AI 選定理由の補足テキスト（グレー・小文字） |
| `button` | `fit-content` 幅・primary 色・disabled 時グレー |
| `.success` | 成功メッセージ（緑系色） |
| `.error` | エラーメッセージ（赤系色） |

## 5. コンポーネント間データフロー

```
SCR-001 (overview/+page.svelte)
  |
  | navigate: /projects/setup?overviewId={id}
  v
SCR-002 (projects/setup/+page.svelte)
  |-- onMount --> URL から overviewId 取得
  |-- $effect  --> handleExtract() 自動実行
  |               |-- POST /api/v1/projects/extract-features --> features, components
  |               |-- POST /api/v1/projects/suggest-name    --> candidates, candidateItems
  |
  |-- ユーザ操作: プロジェクト名選択・localPath 編集
  |
  |-- handleInitDirectory()
  |               |-- POST /api/v1/projects/init-directory  --> directoryStatus
  |
  | back-link: /overview?overviewId={overviewId}
  v
SCR-001 (overview/+page.svelte) ← overviewId 引数で概要内容を復元
```

## 6. 今後の改善候補

| # | 課題 | 対応案 |
| --- | --- | --- |
| 1 | `BASE_DIR` がハードコード | 環境変数 `VITE_DEFAULT_BASE_DIR` として外部設定化 |
| 2 | 機能抽出結果の編集不可 | features / components を編集可能なリストに変更 |
| 3 | ディレクトリ作成後の次ステップ誘導 | 作成成功後に SCR-003（GitHub リポジトリ作成）へ自動遷移するリンクを追加 |
| 4 | コンボボックス UX | `<select>` と自由入力を切り替えるトグル UI を検討 |
