## 2. フロントバリデーションエラー

**E-01: 空文字入力** / **E-02: 文字数超過**

```mermaid
sequenceDiagram
    actor User as ユーザ
    participant FE as フロントエンド<br>(SCR-001)

    User->>FE: 「保存する」クリック
    activate FE
    FE->>FE: handleSubmit() 実行
    FE->>FE: validate() 呼び出し
    alt E-01: content.trim() が空
        FE-->>User: errorMessage = "システム概要を入力してください。"<br>（API 呼び出しなし）
    else E-02: content.length > 4096
        FE-->>User: errorMessage = "4096文字以内で…（現在: N文字）。"<br>（API 呼び出しなし）
    end
    deactivate FE
```

## 3. API バリデーションエラー（422）

**E-03**: フロントのバリデーションを通過したが API 側でバリデーション違反

```mermaid
sequenceDiagram
    actor User as ユーザ
    participant FE as フロントエンド<br>(SCR-001)
    participant API as musuhi-api<br>(Handler)
    participant SVC as Service層

    User->>FE: 「保存する」クリック
    FE->>FE: validate() → OK
    FE->>API: POST /api/v1/system-overviews<br>{"content": "..."}
    activate API
    API->>SVC: Create(content)
    activate SVC
    SVC->>SVC: バリデーション判定<br>(ErrValidation)
    SVC-->>API: fmt.Errorf("%w: ...", ErrValidation)
    deactivate SVC
    API->>API: errors.Is(err, ErrValidation) → true
    API-->>FE: HTTP 422<br>{"error": {"code": "VALIDATION_ERROR", "message": "..."}}
    deactivate API
    FE->>FE: errorMessage = data.error.message
    FE-->>User: エラーメッセージ表示
```

## 4. リソース未存在エラー（404）

**E-04**: GET /{id} で指定 ID のレコードが存在しない（将来実装を含む）

```mermaid
sequenceDiagram
    actor User as ユーザ
    participant FE as フロントエンド
    participant API as musuhi-api<br>(Handler)
    participant SVC as Service層
    participant REPO as Repository層
    participant DB as PostgreSQL

    User->>FE: GET /overview/{id} アクセス
    FE->>API: GET /api/v1/system-overviews/{id}
    activate API
    API->>SVC: GetByID(id)
    activate SVC
    SVC->>SVC: uuid.Parse(id) → OK
    SVC->>REPO: FindByID(id)
    activate REPO
    REPO->>DB: SELECT * FROM system_overviews WHERE id = $1
    DB-->>REPO: pgx.ErrNoRows（0件）
    REPO-->>SVC: fmt.Errorf("...: %w", pgx.ErrNoRows)
    deactivate REPO
    SVC->>SVC: errors.Is(err, pgx.ErrNoRows) → ErrNotFound に変換
    SVC-->>API: fmt.Errorf("%w: id=%s", ErrNotFound, id)
    deactivate SVC
    API->>API: errors.Is(err, ErrNotFound) → true
    API-->>FE: HTTP 404<br>{"error": {"code": "NOT_FOUND", "message": "..."}}
    deactivate API
    FE-->>User: エラーメッセージ表示
```

## 5. JSON 形式不正エラー（400）

**E-05**: リクエストボディが JSON として解析できない

```mermaid
sequenceDiagram
    actor Client as クライアント
    participant API as musuhi-api<br>(Handler)

    Client->>API: POST /api/v1/system-overviews<br>Body: {invalid json}
    activate API
    API->>API: json.NewDecoder(r.Body).Decode(&req)
    API->>API: デコードエラー発生
    API-->>Client: HTTP 400<br>{"error": {"code": "BAD_REQUEST", "message": "リクエスト形式が不正です"}}
    deactivate API

    Note over API: Service は呼び出されない
```

## 6. DB 接続エラー / タイムアウト（500）

**E-06**: Repository がDB接続に失敗する

```mermaid
sequenceDiagram
    actor User as ユーザ
    participant FE as フロントエンド<br>(SCR-001)
    participant API as musuhi-api<br>(Handler)
    participant SVC as Service層
    participant REPO as Repository層
    participant DB as PostgreSQL

    User->>FE: 「保存する」クリック
    FE->>API: POST /api/v1/system-overviews<br>{"content": "..."}
    activate API
    API->>SVC: Create(content)
    activate SVC
    SVC->>SVC: バリデーション → OK
    SVC->>REPO: Create(systemOverview)
    activate REPO
    REPO->>DB: INSERT INTO system_overviews ...
    DB-->>REPO: 接続エラー（pgx エラー）
    REPO-->>SVC: fmt.Errorf("systemOverviewRepository.Create: %w", err)
    deactivate REPO
    SVC-->>API: fmt.Errorf("systemOverviewService.Create: %w", err)
    deactivate SVC
    API->>API: errors.Is → ErrValidation/ErrNotFound でない → 500
    API-->>FE: HTTP 500<br>{"error": {"code": "INTERNAL_ERROR", "message": "サーバーエラーが発生しました"}}
    deactivate API
    FE->>FE: errorMessage = "サーバーエラーが発生しました。…"
    FE-->>User: エラーメッセージ表示（内部エラー詳細は非表示）
```

## 7. ネットワーク断（フロント→API）

**E-07**: `fetch()` が例外をスローする（接続タイムアウト・CORS・オフライン等）

```mermaid
sequenceDiagram
    actor User as ユーザ
    participant FE as フロントエンド<br>(SCR-001)
    participant NW as ネットワーク

    User->>FE: 「保存する」クリック
    FE->>FE: validate() → OK
    FE->>FE: isSubmitting = true
    FE->>NW: fetch POST /api/v1/system-overviews
    activate NW
    NW--xFE: TypeError: Failed to fetch<br>（接続失敗 / タイムアウト / CORS エラー等）
    deactivate NW
    FE->>FE: catch 句 → errorMessage = "ネットワークエラーが発生しました。接続を確認してください。"
    FE->>FE: finally → isSubmitting = false
    FE-->>User: エラーメッセージ表示
```

## 8. DB タイムアウト

**E-08**: DB クエリが完了する前にタイムアウトが発生する（`context.DeadlineExceeded`）

```mermaid
sequenceDiagram
    actor User as ユーザ
    participant FE as フロントエンド<br>(SCR-001)
    participant API as musuhi-api<br>(Handler)
    participant REPO as Repository層
    participant DB as PostgreSQL

    User->>FE: 「保存する」クリック
    FE->>API: POST /api/v1/system-overviews
    activate API
    API->>REPO: Create(systemOverview)
    activate REPO
    REPO->>DB: INSERT INTO system_overviews ...
    Note over DB: クエリ処理中...
    Note over REPO: context.DeadlineExceeded<br>（タイムアウト）
    REPO-->>API: fmt.Errorf("...: %w", context.DeadlineExceeded)
    deactivate REPO
    API->>API: 500 に分類（ErrValidation/ErrNotFound 以外）
    API-->>FE: HTTP 500<br>{"error": {"code": "INTERNAL_ERROR", ...}}
    deactivate API
    FE-->>User: エラーメッセージ表示

    Note over API: 現状: context タイムアウト設定なし<br>（将来: context.WithTimeout を推奨）
```

> **補足**: 現状の実装では `context.WithTimeout` を使用していないため、DB タイムアウトはクライアント側のタイムアウトまたはDB側の設定に依存する。将来的に Handler でリクエストコンテキストにタイムアウトを設定することを推奨する。
