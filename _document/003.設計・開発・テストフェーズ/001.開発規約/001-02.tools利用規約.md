## 1. 目的

本ドキュメントは、tools ディレクトリ配下に追加するスクリプトの配置方法と、
README の記載ルールを統一することを目的とする。
スクリプト単体ではなく「用途・実行方法・前提条件」を必ずセットで管理し、
保守性と再利用性を担保する。

## 2. 適用範囲

- 対象ディレクトリ: Musuhi 配下の tools ディレクトリ
- 対象成果物: shell script、補助スクリプト、実行バッチ、CLI 呼び出し用ラッパー
- 対象者: tools 配下に新規追加または更新を行う作業者

## 3. ディレクトリ構成ルール

tools 配下にスクリプトを追加する場合、直下にファイルを置いてはならない。
必ず tools/{スクリプト名}/ ディレクトリを作成し、その中にスクリプト本体を配置する。

標準構成:

```text
tools/
  {スクリプト名}/
    README.md
    {スクリプト名}.sh
```

補助ファイルが必要な場合は、同一ディレクトリ配下に配置する。

```text
tools/
  {スクリプト名}/
    README.md
    {スクリプト名}.sh
    template.json
    sample.env
```

命名ルール:

- ディレクトリ名はスクリプトの用途が分かる英小文字とアンダースコアを基本とする
- 実行ファイル名はディレクトリ名と合わせる
- 単発用途でも README.md を省略してはならない

## 4. README記載ルール

各 tools/{スクリプト名}/ ディレクトリの直下には必ず README.md を配置し、
少なくとも以下を記載する。

- 使用目的: 何のためのスクリプトか
- 使用方法: 実行コマンド、引数、実行場所
- 前提条件: 必要なツール、認証、環境変数、依存ファイル
- 入出力: 更新対象、生成物、外部影響
- 注意事項: 破壊的操作の有無、再実行可否、運用上の注意

README の推奨テンプレート:

```markdown
# {スクリプト名}

## 目的

## 前提条件

## 使用方法

## 入出力

## 注意事項
```

## 5. スクリプト作成・更新ルール

- 新規スクリプト追加時は、スクリプト本体と README.md を同一変更で追加する
- 新規スクリプト追加時は、tools 直下の README.md のインデックスとツール一覧も同一変更で更新する
- 既存スクリプトを改修する場合も、README.md の内容を必要に応じて更新する
- スクリプトが外部サービスや認証情報を利用する場合は、README.md に前提条件を明記する
- 利用手順が複数ステップに分かれる場合は、README.md に順序を明記する
- 将来再利用する可能性がある処理は、単発コマンド履歴ではなく tools 配下の規定構成に格納する
- 可視化系ツールを追加する場合は、依存のインストール手順を README.md に明記する
- Forgejo Projects のデータを利用する可視化ツールは、`gh api` で一覧取得して生成する手順を標準化する
- 難易度可視化を行う場合は、`Estimate` から数値難易度への変換表（例: `XS=1, S=2, M=3, L=5, XL=8`）を README.md に明記する
- タスク完了ごとに成果物を出力する運用とし、出力タイミング・出力先・命名規則を README.md に明記する
- タスク完了ごとに出力ファイルを生成するツールは、同一タスク内で上書きせず時刻入りファイル名で保存する
- 進捗可視化HTMLの標準命名は `{プロジェクト名}進捗_<yyyymmddhhmmss>.html` とする

既存の tools 直下スクリプトは、新規追加時のルールに合わせることを原則とし、
今後改修するタイミングで段階的に規定構成へ移行する。

## 6. 禁止事項

- tools 直下に README.md なしでスクリプトファイルを直接配置すること
- ディレクトリ名と無関係な名前のスクリプトを配置すること
- 使用目的や実行方法が不明なままスクリプトを追加すること
- 実行に認証や環境変数が必要なのに README.md に未記載のままにすること

## 7. 運用例

例: prompting log 保存スクリプトを追加する場合

```text
tools/
  save_promptinglog/
    README.md
    save_promptinglog
```

README.md には、たとえば以下を記載する。

- 目的: prompting log を所定の保存先へ退避する
- 使用方法: 作業ディレクトリと実行コマンド
- 前提条件: 実行権限、依存コマンド、保存先ディレクトリ
- 注意事項: 上書き有無、既存ログへの影響

例: Issue 難易度ロードマップを生成する場合

```text
tools/
  generate_issue_roadmap/
    README.md
    go.mod
    main.go
    source/
      app.go
```

README.md には、たとえば以下を記載する。

- 目的: Forgejo ProjectsのIssue一覧を難易度軸で可視化する
- 前提条件: `gh` 認証、Go 1.22+
- 使用方法: `go run . -owner <owner> -repo <repo> -project-number <n>` の実行コマンド
- 注意事項: 件数上限、難易度マッピング、出力先
