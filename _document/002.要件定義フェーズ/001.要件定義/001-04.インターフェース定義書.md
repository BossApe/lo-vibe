## 1. 目的

本ドキュメントは、Musuhi API の外部インターフェースを定義し、フロントエンド・外部ツールとの接続仕様を確立する。

## 2. API 設計方針

- RESTful 設計原則に準拠する
- ベース URL: `https://{host}/api/v1`
- リクエスト・レスポンスは JSON 形式とする
- 認証は Bearer トークン（JWT）を使用する
- HTTP ステータスコードを適切に返す
- エラーレスポンスは共通フォーマットとする

## 3. 共通仕様

### 3.1 認証

```
Authorization: Bearer {JWT_TOKEN}
```

- 認証不要なエンドポイント: `POST /auth/login`, `GET /health`
- それ以外は全エンドポイントで認証必須

### 3.2 リクエスト・レスポンス形式

```
Content-Type: application/json
Accept: application/json
```

### 3.3 エラーレスポンス形式

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "プロジェクト名は必須です",
    "details": [
      { "field": "name", "message": "required" }
    ]
  }
}
```

| エラーコード | HTTP ステータス | 説明 |
| --- | --- | --- |
| UNAUTHORIZED | 401 | 認証エラー |
| FORBIDDEN | 403 | 権限エラー |
| NOT_FOUND | 404 | リソース未存在 |
| VALIDATION_ERROR | 422 | 入力バリデーションエラー |
| CONFLICT | 409 | 競合エラー（重複など） |
| INTERNAL_ERROR | 500 | サーバー内部エラー |

### 3.4 ページネーション

リスト系 API は以下のクエリパラメータをサポートする。

| パラメータ | 型 | デフォルト | 説明 |
| --- | --- | --- | --- |
| page | integer | 1 | ページ番号 |
| limit | integer | 20 | 1ページあたりの件数（最大100） |

レスポンスに `pagination` オブジェクトを付与する。

```json
{
  "data": [...],
  "pagination": {
    "total": 100,
    "page": 1,
    "limit": 20,
    "totalPages": 5
  }
}
```

## 4. API 詳細仕様

### 4.1 プロジェクト系 API

#### POST /projects — プロジェクト作成

**リクエスト:**
```json
{
  "name": "string (必須, max:128)",
  "description": "string (任意, max:1024)",
  "startDate": "2026-05-01"
}
```

**レスポンス (201 Created):**
```json
{
  "data": {
    "id": "uuid",
    "name": "string",
    "description": "string",
    "startDate": "2026-05-01",
    "status": "active",
    "createdAt": "2026-05-01T00:00:00Z"
  }
}
```

#### GET /projects — プロジェクト一覧

**クエリパラメータ:** `page`, `limit`, `status` (active / archived)

**レスポンス (200 OK):**
```json
{
  "data": [ { ...プロジェクトオブジェクト } ],
  "pagination": { ... }
}
```

#### GET /projects/{id} — プロジェクト詳細

**レスポンス (200 OK / 404 Not Found)**

### 4.2 セッション・プロンプトログ系 API

#### POST /sessions — セッション作成

**リクエスト:**
```json
{
  "projectId": "uuid (必須)",
  "title": "string (任意, max:256)"
}
```

**レスポンス (201 Created):**
```json
{
  "data": {
    "id": "uuid",
    "projectId": "uuid",
    "title": "string",
    "createdAt": "2026-05-01T00:00:00Z"
  }
}
```

#### POST /sessions/{id}/messages — メッセージ追加

**リクエスト:**
```json
{
  "role": "user | assistant",
  "content": "string (必須)",
  "timestamp": "2026-05-01T00:00:00Z"
}
```

#### GET /sessions — セッション一覧

**クエリパラメータ:** `projectId` (必須), `page`, `limit`

#### GET /sessions/{id} — セッション詳細（メッセージ含む）

---

### 4.3 文書管理系 API

#### POST /documents — 文書作成

**リクエスト:**
```json
{
  "projectId": "uuid (必須)",
  "title": "string (必須, max:128)",
  "content": "string (Markdown, 必須)",
  "comment": "string (任意, max:256)"
}
```

**レスポンス (201 Created):** 文書オブジェクト

#### PUT /documents/{id} — 文書更新

**リクエスト:** `title`, `content`, `comment`

#### GET /documents/{id} — 文書詳細

#### GET /documents/{id}/history — 変更履歴

**レスポンス (200 OK):**
```json
{
  "data": [
    {
      "version": 3,
      "content": "string",
      "comment": "string",
      "updatedAt": "2026-05-01T00:00:00Z"
    }
  ]
}
```

#### GET /documents/{id}/summary — 文書要約取得

**レスポンス (200 OK):**
```json
{
  "data": {
    "documentId": "uuid",
    "version": 3,
    "summary": "要点要約テキスト",
    "generatedAt": "2026-05-01T00:00:00Z"
  }
}
```

#### POST /documents/{id}/approve — 文書承認

**リクエスト:**
```json
{
  "approved": true,
  "comment": "要件定義レビュー承認"
}
```

**レスポンス (200 OK):**
```json
{
  "data": {
    "documentId": "uuid",
    "approvalStatus": "approved",
    "approvedBy": "uuid",
    "approvedAt": "2026-05-01T00:00:00Z"
  }
}
```

### 4.4 aider 連携 API

#### POST /tasks/aider — aider 実行

**リクエスト:**
```json
{
  "projectId": "uuid (必須)",
  "taskDescription": "string (必須)",
  "repositoryUrl": "string (必須)",
  "templateId": "uuid (任意)"
}
```

**レスポンス (202 Accepted):**
```json
{
  "data": {
    "taskId": "uuid",
    "status": "pending"
  }
}
```

#### GET /tasks/{id} — タスク状態確認

**レスポンス (200 OK):**
```json
{
  "data": {
    "taskId": "uuid",
    "status": "running | completed | failed",
    "diff": "string (unified diff 形式)",
    "log": "string"
  }
}
```

### 4.5 進捗管理 API

#### GET /projects/{id}/progress — プロジェクト進捗取得

**レスポンス (200 OK):**
```json
{
  "data": {
    "projectId": "uuid",
    "milestones": [
      {
        "id": "MS-001",
        "name": "Phase 0 完了",
        "status": "in_progress",
        "tasks": [
          { "frId": "FR-001", "status": "completed" },
          { "frId": "FR-002", "status": "in_progress" }
        ],
        "completionRate": 50
      }
    ]
  }
}
```

#### GET /projects/{id}/sync-status — 外部同期状態取得

**レスポンス (200 OK):**
```json
{
  "data": {
    "projectId": "uuid",
    "issuesSync": "in_sync",
    "projectsSync": "in_sync",
    "rulesSync": "partial",
    "lastSyncedAt": "2026-05-01T00:00:00Z"
  }
}
```

### 4.6 テンプレート系 API

#### GET /templates — テンプレート一覧

**クエリパラメータ:** `phase` (phase0 / phase1 / phase2), `category`

#### POST /templates/{id}/generate — 指示文生成

**リクエスト:**
```json
{
  "parameters": {
    "projectName": "string",
    "taskDescription": "string"
  }
}
```

**レスポンス (200 OK):**
```json
{
  "data": {
    "generatedText": "string"
  }
}
```

---

### 4.7 レポート系 API

#### POST /reports/generate — レポート生成

**リクエスト:**
```json
{
  "projectId": "uuid (必須)",
  "startDate": "2026-05-01",
  "endDate": "2026-05-31",
  "format": "markdown | pdf"
}
```

**レスポンス (202 Accepted):** タスクID を返す

---

### 4.8 レガシー改修支援 API

#### POST /legacy/analyze — レガシー分析実行

**リクエスト:**
```json
{
  "projectId": "uuid (必須)",
  "analysisTargets": ["document", "bug", "security", "test", "iac"],
  "sourceFiles": ["base64 エンコードファイル配列 または ファイルパス"]
}
```

**レスポンス (202 Accepted):** タスクID を返す

#### GET /tasks/{id}/legacy-result — 分析結果取得

**レスポンス (200 OK):**
```json
{
  "data": {
    "missingDocuments": [...],
    "improvements": [...],
    "issues": [...],
    "testCode": "string",
    "testSpec": "string",
    "coverageReport": "string",
    "iacTemplates": [...]
  }
}
```
