## 1. 目的

本ドキュメントは、Musuhi API の外部インターフェースを定義し、フロントエンド・外部ツールとの接続仕様を確立する。

## 2. API 設計方針

- RESTful 設計原則に準拠する
- ベース URL: `https://{host}/api/v1`
- リクエスト・レスポンスは JSON 形式とする
- 認証は Bearer トークン（JWT）を使用する
- HTTP ステータスコードを適切に返す
- エラーレスポンスは共通フォーマットとする

## 3. 共通仕様

### 3.1 認証

```
Authorization: Bearer {JWT_TOKEN}
```

- 認証不要なエンドポイント: `POST /auth/login`, `GET /health`
- それ以外は全エンドポイントで認証必須

### 3.2 リクエスト・レスポンス形式

```
Content-Type: application/json
Accept: application/json
```

### 3.3 エラーレスポンス形式

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "プロジェクト名は必須です",
    "details": [
      { "field": "name", "message": "required" }
    ]
  }
}
```

| エラーコード | HTTP ステータス | 説明 |
| --- | --- | --- |
| UNAUTHORIZED | 401 | 認証エラー |
| FORBIDDEN | 403 | 権限エラー |
| NOT_FOUND | 404 | リソース未存在 |
| VALIDATION_ERROR | 422 | 入力バリデーションエラー |
| CONFLICT | 409 | 競合エラー（重複など） |
| INTERNAL_ERROR | 500 | サーバー内部エラー |

### 3.4 ページネーション

リスト系 API は以下のクエリパラメータをサポートする。

| パラメータ | 型 | デフォルト | 説明 |
| --- | --- | --- | --- |
| page | integer | 1 | ページ番号 |
| limit | integer | 20 | 1ページあたりの件数（最大100） |

レスポンスに `pagination` オブジェクトを付与する。

```json
{
  "data": [...],
  "pagination": {
    "total": 100,
    "page": 1,
    "limit": 20,
    "totalPages": 5
  }
}
```

## 4. API 詳細仕様

### 4.1 システム概要系 API（FR-001）

#### POST /system-overviews — システム概要を保存

**リクエスト:**
```json
{
  "content": "string (必須, max:4096)"
}
```

**レスポンス (201 Created):**
```json
{
  "data": {
    "id": "uuid",
    "content": "string",
    "createdAt": "2026-05-01T00:00:00+09:00"
  }
}
```

#### GET /system-overviews/{id} — システム概要を取得

**レスポンス (200 OK / 404 Not Found)**

---

#### PUT /system-overviews/{id} — システム概要を更新

**パスパラメータ:** `id` (UUID v4, 必須)

**リクエスト:**
```json
{
  "content": "string（必須, 1〜4096文字）"
}
```

**レスポンス (200 OK):**
```json
{
  "data": {
    "id": "uuid",
    "content": "string",
    "createdAt": "RFC 3339"
  }
}
```

**エラー:** 400 / 404 / 422 / 500

---

### 4.2 プロジェクト系 API（FR-002～FR-004）

#### POST /projects/extract-features — 機能・構成要素を抽出

**リクエスト:**
```json
{
  "overviewId": "uuid (必須)"
}
```

**レスポンス (200 OK):**
```json
{
  "data": {
    "features": ["string"],
    "components": ["string"]
  }
}
```

#### POST /projects/suggest-name — プロジェクト名候補を生成

**リクエスト:**
```json
{
  "overviewId": "uuid (必須)"
}
```

**レスポンス (200 OK):**
```json
{
  "data": {
    "candidates": ["string"],
    "items": [
      {
        "name": "string",
        "reason": "string",
        "aiSuggested": true
      }
    ]
  }
}
```

#### POST /projects/init-directory — 初期ディレクトリ構造を作成

**リクエスト:**
```json
{
  "projectName": "string (必須)",
  "localPath": "string (必須, 最終作成先絶対パス)",
  "template": "default"
}
```

**レスポンス (201 Created):**
```json
{
  "data": { "id": "uuid", "directoryStatus": "success" }
}
```

#### POST /projects/with-external — Forgejoリポジトリを作成・commit/push

**リクエスト:**
```json
{
  "projectId": "uuid (必須)",
  "owner": "string (必須)",
  "visibility": "public | private",
  "commitMessage": "string (必須)"
}
```

**レスポンス (201 Created):**
```json
{
  "data": {
    "repositoryUrl": "https://github.com/...",
    "externalProjectId": "string",
    "pushStatus": "success"
  }
}
```

#### POST /projects/{id}/external-setup — 外部側初期構造を作成

**レスポンス (200 OK)**

#### POST /projects/{id}/forgejo-projects — Forgejo Projects ボードを作成

**リクエスト:**
```json
{
  "projectId": "uuid (必須)",
  "repositoryUrl": "string (必須)"
}
```

**レスポンス (201 Created):**
```json
{
  "data": { "forgejoProjectsUrl": "https://github.com/orgs/..." }
}
```

#### POST /projects/{id}/phase0-tasks — Phase0 固定タスクを生成・登録

**レスポンス (201 Created):**
```json
{
  "data": {
    "tasks": [ { "id": "TK0-1-1", "title": "string", "status": "open" } ],
    "registerStatus": "success"
  }
}
```

### 4.3 ドキュメント自動生成系 API（FR-005 / FR-007 / FR-009）

#### POST /documents/generate-proposal — 提案・要求仕様書を生成

**リクエスト:**
```json
{
  "overviewId": "uuid (必須)",
  "features": ["string (必須)"]
}
```

**レスポンス (202 Accepted):**
```json
{
  "data": { "taskId": "uuid", "status": "pending" }
}
```

#### POST /documents/generate-requirements — 要件定義書を生成

**リクエスト:**
```json
{
  "proposalDocumentId": "uuid (必須, 承認済み)"
}
```

**レスポンス (202 Accepted):** タスクID を返す

#### POST /documents/generate-conventions — 開発規約を生成

**リクエスト:**
```json
{
  "projectId": "uuid (必須)",
  "techStack": ["string (必須)"]
}
```

**レスポンス (202 Accepted):** タスクID を返す

#### POST /projects/{id}/progress-file — 進捗ファイルを出力・commit/push

**リクエスト:**
```json
{
  "taskId": "string (必須, 完了タスクID)"
}
```

**レスポンス (200 OK):**
```json
{
  "data": { "filePath": "string", "pushStatus": "success" }
}
```

### 4.4 ドキュメントレビュー・承認系 API（FR-006）

#### GET /documents/{id}/summary — 文書要約を取得

**レスポンス (200 OK):**
```json
{
  "data": {
    "documentId": "uuid",
    "version": 3,
    "summary": "要点要約テキスト",
    "generatedAt": "2026-05-01T00:00:00+09:00"
  }
}
```

#### PUT /documents/{id} — 文書を更新（リファクタリング）

**リクエスト:** `title`, `content`, `comment`

#### POST /documents/{id}/approve — 文書承認

**リクエスト:**
```json
{
  "approved": true,
  "comment": "string"
}
```

**レスポンス (200 OK):**
```json
{
  "data": {
    "documentId": "uuid",
    "approvalStatus": "approved",
    "approvedBy": "uuid",
    "approvedAt": "2026-05-01T00:00:00+09:00"
  }
}
```

---

### 4.5 タスク分割系 API（FR-008）

#### POST /projects/{id}/task-breakdown — Phase/Sprint/Ticket 分割を実施

**リクエスト:**
```json
{
  "requirementsDocumentId": "uuid (必須, 承認済み)",
  "phaseCount": 2,
  "sprintsPerPhase": 3
}
```

**レスポンス (200 OK):**
```json
{
  "data": {
    "phases": [
      {
        "id": "Phase0",
        "sprints": [
          { "id": "Sprint1", "tickets": [ { "id": "TK1-1-1", "title": "string" } ] }
        ]
      }
    ]
  }
}
```

#### POST /projects/{id}/register-tasks — タスクをForgejo Projects へ登録

**リクエスト:**
```json
{
  "phases": [ "...分割結果..." ]
}
```

**レスポンス (200 OK):**
```json
{
  "data": { "registerStatus": "success", "registeredCount": 12 }
}
```

### 4.7 Ticket・aider 実行系 API（FR-011）

#### POST /tickets/{id}/start-cycle — Ticket 開発サイクルを開始

**リクエスト:**
```json
{
  "repositoryUrl": "string (必須)"
}
```

**レスポンス (200 OK):**
```json
{
  "data": { "ticketId": "uuid", "status": "in_progress" }
}
```

#### POST /tasks/aider — aider 連携で実装差分を生成

**リクエスト:**
```json
{
  "projectId": "uuid (必須)",
  "taskDescription": "string (必須)",
  "repositoryUrl": "string (必須)",
  "templateId": "uuid (任意)"
}
```

**レスポンス (202 Accepted):**
```json
{
  "data": { "taskId": "uuid", "status": "pending" }
}
```

#### GET /tasks/{id} — タスク状態確認

**レスポンス (200 OK):**
```json
{
  "data": {
    "taskId": "uuid",
    "status": "running | completed | failed",
    "diff": "string (unified diff 形式)",
    "log": "string"
  }
}
```

#### POST /tickets/{id}/complete — Ticket を完了に更新

**リクエスト:**
```json
{
  "testResults": { "unit": "pass", "integration": "pass", "acceptance": "pass" }
}
```

**レスポンス (200 OK):**
```json
{
  "data": { "ticketId": "uuid", "status": "done" }
}
```
