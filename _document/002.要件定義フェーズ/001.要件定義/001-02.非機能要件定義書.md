## 1. 目的

本ドキュメントは、001.提案・要求仕様フェーズの非機能要件 (003-07) を要件定義フェーズで詳細化し、設計・実装・テスト・運用の基準を確立する。

## 2. 適用範囲

- Phase 0 (MVP) から Phase 2 まで
- Musuhi アプリケーション層およびインフラ層

## 5. セキュリティ要件

| NFR-ID | 項目 | 要件内容 | 実装指針 |
| --- | --- | --- | --- |
| NFR-SEC-001 | 認証 | 管理系エンドポイントは認証必須 | JWT または OAuth2 を使用 |
| NFR-SEC-002 | 機密情報管理 | .env への平文保存禁止 | Docker Secret / 環境変数を利用 |
| NFR-SEC-003 | アクセス制御 | 役割別アクセス制御を定義（PO / 開発者 / 閲覧者） | ミドルウェアで一元制御 |
| NFR-SEC-004 | ログ保護 | 個人情報・機密情報はログ保存前にマスク | ログフィルタ実装 |
| NFR-SEC-005 | 通信暗号化 | Traefik による TLS 終端 | HTTPS 強制 |
| NFR-SEC-006 | 入力バリデーション | すべての API 入力を検証 | OWASP Top 10 準拠 |
| NFR-SEC-007 | 依存ライブラリ脆弱性 | 既知脆弱性ゼロを維持 | CI での Trivy / govulncheck 実行 |

## 6. 保守性・拡張性要件

| NFR-ID | 項目 | 要件内容 |
| --- | --- | --- |
| NFR-MAIN-001 | モジュール分離 | 機能単位でモジュールを分離し、依存関係を明示する |
| NFR-MAIN-002 | テストカバレッジ | 80% 以上を維持する |
| NFR-MAIN-003 | コーディング規約 | 005-01.コーディング規約に準拠する |
| NFR-MAIN-004 | ドキュメント同期 | master マージ前に仕様文書を更新する |
| NFR-EXT-001 | LLM 交換容量 | LiteLLM 経由の統一エンドポイントでモデル交換可能にする |
| NFR-EXT-002 | ストレージ拡張 | Garage (S3 互換) への成果物保管に対応する |
| NFR-EXT-003 | Kubernetes 移行 | Docker Compose サービスの境界を Kubernetes 移行可能な設計にする |
