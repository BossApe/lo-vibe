## 1. 目的

本ドキュメントは、Musuhi の性能、可用性、セキュリティ、運用に関する基準を定義する。

## 2. 適用範囲

- Phase 0 (MVP) から Phase 2 まで
- アプリケーション層およびインフラ層

## 3. SLO / 性能要件

| NFR-PERF-001 | 文書検索応答 | 1 秒以内 (p95) | アプリログ計測 | FR-003 |
| NFR-PERF-002 | プロンプト生成応答 | 3 秒以内 (p95) | API 応答時間計測 | FR-002/FR-004 |
| NFR-PERF-003 | 進捗表示更新 | 2 秒以内 | 画面更新計測 | FR-005 |

## 4. 可用性 / 信頼性要件

| NFR-ID | 項目 | 目標値 | 測定方法 |
| --- | --- | --- | --- |
| NFR-AVL-001 | サービス稼働率 | 99.0% 以上 | 月次稼働集計 |
| NFR-REL-001 | ログ永続性 | セッションログ欠損 0 件 | 復旧テスト |
| NFR-REL-002 | バックアップ成功率 | 100% | 日次ジョブ監視 |

## 5. セキュリティ要件

| NFR-ID | 項目 | 要件 |
| --- | --- | --- |
| NFR-SEC-001 | 認証 | 管理系エンドポイントは認証必須 |
| NFR-SEC-002 | 機密情報 | .env に平文保存しない (secret 利用) |
| NFR-SEC-003 | アクセス制御 | 役割別アクセス制御を定義 |
| NFR-SEC-004 | ログ保護 | 個人情報は保存前にマスク |
